Where am I?

I created the game.

dragonballplay.com

State idea

  • state definition - class with 4 methods: preload, create, update, render and optionally init
  • only one state is active
  • gameplay of the game is to switch to state, ex. choose lang, select character form menu, fight, defeat or win screen

init method

  • uruchamia się gdy stan zostanie aktywowany - jako pierwsza metoda

preload method

  • metoda w której ściągamy assety (pliki json, pliki z ścieżkami dźwiękowymi)

create method

  • uruchamiana raz podczas tworzenie stanu
  • w niej tworzymy wszystkie obiektu występujące w tym stanie gry

update method

  • Phaser.Game#raf via requestAnimationFrame albo setTimeout
  • 38881: The core game loop - method: Phaser.Game#update

render method

  • metoda która uruchamia się po cały procesie aktualizacji obiektów w Canvas
  • w niej możemy włączyć prezentację technicznych detali naszych obiektów
  • wykonuje się zawsze po metodzie update

Day I: Where do we start?

31↑

Use ECMAScript 6.
webpack i Babel.js take care for compiling to ECMAScript 5 (for browser).

webpack.config.js

                module.exports = {
                    resolve: { extensions: ['.es6.js', '.js', ''] },
                    entry: './app/scripts/main',
                    output: {
                        filename: 'bundle.js',
                        path: './app/dist'
                    },
                    module: {
                        loaders: [{
                            test: /\.es6\.js/,
                            exclude: /node_modules/,
                            loader: 'babel?stage=0'
                        }]
                    }
                };

Advantages of ECMAScript 6

  • modules - no global variables
  • importing - require only dependency
  • smooth syntax of class definition
  • "simple" inheritance
  • default parameters
  • arrow function
  • destructuring assignment
                var [x, y] = ['abc', 100]
                // x = 'abc', y = 100

dodajemy smocze kule

Tworzymy na ich potrzebę grupę, aby można było łatwo tworzyć kolizje

Nawigacja

  • if (keyboard.isDown(Phaser.Keyboard.LEFT) { player.x -= 5; }
  • ważne, że aby była kolizja gracza z czymkolwiek, to musimy operować jego prędkością a nie tylko pozycją
  • musimy player.velocity.x -= 5
  • trzeba tym samym czyścić przy każdej iteracji game loopa player.body.velocity.x = 0;

Kolizje - temat rzeka

- player - // włączamy arcade mode dla naszego zawodnika - this.physics.arcade.enable(player) - mapa - map.setCollisionByIndex(1) // numerowanie o 1 - layer = map.createLayer(name) - this.physics.arcade.collide(player, layer) - od teraz player będzie kolidować z kafelkiem o indeksie 0

Zbieranie elementów - smoczych kul

- grupa musi mieć ciało - balls = this.add.group() - balls.enableBody = true; - grupa musi mieć zdefiniowaną fizykę - this.physics.arcade.enable(balls); - this.physics.arcade.collide(player, balls, (player, ball) => {})
